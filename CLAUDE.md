# CLAUDE.md

Guidance for AI assistants (and humans) working in this repo.

## Project

`utexas.network` — an open-source webring/directory for UT Austin students, builders, and campus
projects. This repo contains both the public Next.js site **and** the embeddable webring widget that
members add to their own sites.

## Stack & conventions

- **Next.js 16 (App Router), React 19, TypeScript.** Path alias `@/*` → `src/*`.
- **Linting:** ESLint only (`npm run lint`). No Prettier — match surrounding style.
- **Scripts:** `npm run dev` / `build` / `start` / `lint`.
- **Naming:**
  - React components: `PascalCase` (e.g. `NetworkGraph.tsx`).
  - Data/util modules: `camelCase` (e.g. `profileImage.ts`).
  - Route folders, data folders, and slugs/IDs: `kebab-case`.
- **Data:** lives in `src/data/` as TS modules (e.g. `members.ts`), imported via `@/data/...`.
  Machine-written data (generated by Actions) should be **JSON**, not TS, to avoid syntax breakage.
- **Automation scripts:** `.mjs` files in `scripts/` are standalone Node — read env, sanitize input,
  write data files, console-log. See `scripts/apply-approved-request.mjs` as the reference shape.
- **Embed widget:** `src/app/embed.js/route.ts` serves a self-contained IIFE. It resolves its base
  URL from the `<script src>` (handles `localhost`), is CORS `*`, and renders the center icon via an
  `<img>` tag. Reuse that pattern rather than inventing new ones.
- **Generated-data → deploy:** the bot-commit-on-diff + Vercel deploy-hook pattern lives in
  `.github/workflows/moderate-join-requests.yml` and `trigger-deploy.yml`. Reuse it for new cron jobs.

## Working agreement (read this every session)

This is a **collaborative** codebase. When working here:

- **Surface pushback** — if a request has a simpler, safer, or more expandable alternative, say so.
- **Propose options** with tradeoffs rather than silently picking one.
- **Ask questions** whenever intent is ambiguous; do not proceed on silent assumptions.
- We converge on solutions through back-and-forth, not one-shot delivery.

## In-flight work

- **UT Tower lighting in the embed** — multi-session, staged effort. The durable tracker and current
  phase status live in [`docs/tower/PLAN.md`](docs/tower/PLAN.md). Read it before starting any
  tower-related session, and update phase status when you close a phase out.
